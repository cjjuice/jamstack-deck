# <Highlight text="J" />-A-M

**JavaScript**: Any dynamic programming during the request/response cycle is handled by JavaScript, running entirely on the client.

---
export default Split

# J-<Highlight text="A" />-M

**APIs**: All server-side functions or databases actions are abstracted into reusable APIs, accessed over HTTPS with JS. Be it SaaS, third-party services or custom-built.

---
export default Split

# J-A-<Highlight text="M" />

**Markup**: Templated markup should be prebuilt at build time, usually using a site generator for content sites, or a build tool for web apps

## JAMStack <Highlight text="Pros" />

<ul>
  <Appear>
    <li>Performance</li>

    <li>Scalability</li>
  </Appear>
</ul>

---
## JAMStack <Highlight text="Cons" />

<ul>
  <Appear>
    <li>Changes in CMS are not immediate, need to build/deploy on every change</li>
    <li>If data changes faster than build time, you will have grab that data client side</li>
    <li>For truly dynamic apps, still will need backend API/ AWS Lambda</li>
    <li>Not traditional app flow, no easy way to transition apps</li>
  </Appear>
</ul>
